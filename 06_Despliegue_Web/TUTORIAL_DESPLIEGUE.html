Despliegue automático del pipeline en la nube

GitHub Pages + GitHub Actions · Predicción de Incumplimiento de Préstamos · EV Parcial 3
0 de 6 pasos

Qué vas a lograr

Publicar un dashboard en vivo en internet (URL pública, gratis, sin tarjeta) que se actualiza solo: cada vez que subes un cambio, GitHub ejecuta tu pipeline de IA en la nube, reentrena los modelos y vuelve a publicar la web con las métricas nuevas.

Tú haces "git push" │ ▼ ┌─────────────────────────── GitHub Actions (nube) ───────────────────────────┐ │ 1. Instala Python → 2. Ejecuta generar_metrics.py (ENTRENA el modelo) │ │ 3. Produce metrics.json → 4. Publica la carpeta site/ en GitHub Pages │ └─────────────────────────────────────────────────────────────────────────────┘ │ ▼ Dashboard público ←──lee── metrics.json (conexión automática)
Los archivos ya están creados en tu proyecto (carpeta 06_Despliegue_Web/ y el workflow .github/workflows/deploy-web.yml). Este tutorial solo te guía para activarlo.
1

Verificar que los archivos existen

Pendiente

Abre una terminal en la carpeta del proyecto y confirma que están los archivos del despliegue:

cd C:\PROJECTOSCLAU\GestionDeDatos
dir 06_Despliegue_Web
dir 06_Despliegue_Web\site
dir .github\workflows

Deberías ver: generar_metrics.py, site\index.html, site\metrics.json y deploy-web.yml.

2

(Opcional) Probarlo en tu computador antes de publicar

Pendiente

Genera las métricas y levanta un servidor local para ver el dashboard:

python 06_Despliegue_Web\generar_metrics.py
cd 06_Despliegue_Web\site
python -m http.server 8000

Abre http://localhost:8000 en tu navegador. Cuando termines, cierra el servidor con Ctrl + C y vuelve a la raíz con cd ..\..

Si abres index.html con doble clic (file://) el navegador bloquea la lectura del JSON. Por eso se usa http.server. En GitHub Pages funciona sin este problema.
3

Subir los archivos a GitHub (commit + push)

Pendiente

Tu repositorio ya está conectado. Sube la carpeta de despliegue y el workflow:

cd C:\PROJECTOSCLAU\GestionDeDatos
git add 06_Despliegue_Web .github/workflows/deploy-web.yml
git commit -m "Despliegue automatico del dashboard en GitHub Pages"
git push origin main
Repositorio destino:
github.com/fergomezs2024/-fernandoGomez_AstridKobrock_EmilioHerrera_Entrega2v
4

Activar GitHub Pages (una sola vez)

Pendiente

Esto le dice a GitHub que publique usando el workflow:

  • Entra a tu repositorio en GitHub.
  • Ve a Settings (Configuración) → menú lateral Pages.
  • En Build and deployment → Source, elige GitHub Actions (NO "Deploy from a branch").
  • Listo, no hay que guardar nada más.
https://github.com/fergomezs2024/-fernandoGomez_AstridKobrock_EmilioHerrera_Entrega2v/settings/pages
Si el repositorio es privado, GitHub Pages gratis requiere que lo pongas en público (Settings → General → abajo "Change visibility"). Para una entrega académica, público está bien.
5

Ver el pipeline ejecutándose en la nube

Pendiente

Cada push dispara el workflow. Míralo en la pestaña Actions:

https://github.com/fergomezs2024/-fernandoGomez_AstridKobrock_EmilioHerrera_Entrega2v/actions
  • Abre el workflow "Desplegar dashboard (Pages)".
  • Verás 2 etapas: build (entrena el modelo) y deploy (publica).
  • En el log de build aparecen tus métricas reales: AUC=0.9703, Recall=0.8880…
Esto es tu evidencia para la rúbrica: el log demuestra que el pipeline se ejecutó en un servidor de la nube (no en tu PC). Toma una captura de pantalla.
6

Abrir tu dashboard público y comprobar la conexión

Pendiente

Cuando la Action termine (punto verde), tu dashboard estará en vivo en:

https://fergomezs2024.github.io/-fernandoGomez_AstridKobrock_EmilioHerrera_Entrega2v/

Prueba la conexión automática (que es el punto clave): cambia algo del pipeline o los datos, vuelve a subir, y verás que la web se actualiza sola sin que toques el HTML.

git commit --allow-empty -m "Forzar reentrenamiento y redepliegue"
git push origin main

En ~1–2 minutos la fecha de "Última actualización" del dashboard cambia: prueba de que la web refleja el último entrenamiento.

🎉 ¡Despliegue completo!

Ya tienes el sistema automático funcionando. Para la defensa puedes mostrar:

  • La URL pública del dashboard (demo en vivo).
  • El log de GitHub Actions entrenando el modelo en la nube (evidencia de "rendimiento/integración nube").
  • El flujo conectado: push → pipeline → metrics.json → web, todo automático.
Esto cierra el indicador de la rúbrica que estaba como "nube-ready": ahora es un despliegue en la nube real y reproducible, con evidencia verificable.

Solución de problemas

  • La Action falla en "build": revisa que 01_Datos_Sucios/loan_data.csv esté subido al repo (el script lo necesita).
  • Sale otra Action roja llamada "deploy-aws": es un workflow viejo de AWS que pide credenciales; puedes ignorarlo o borrar el archivo .github/workflows/deploy-aws.yml.
  • La web carga pero dice "No se pudieron cargar las métricas": espera a que la Action termine; el metrics.json lo genera el propio workflow.
  • 404 al abrir la URL: Pages tarda 1–2 min la primera vez; confirma que en Settings → Pages el Source sea "GitHub Actions".
